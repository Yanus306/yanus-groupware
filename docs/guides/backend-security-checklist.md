# 백엔드 보안 체크리스트

> 프론트 개발과 실서버 점검 과정에서 확인한 **우선 보안 항목**을 정리한 문서입니다.

## 1. 권한 검증 서버 강제

- `PATCH /api/v1/members/{memberId}/role`
- `PATCH /api/v1/members/{memberId}/activate`
- `PATCH /api/v1/members/{memberId}/team`

프론트에서는 관리자, 팀장, 멤버별로 버튼과 라우트를 막아두었지만, 실서버 점검에서는 일부 요청이 서버에서 허용되었습니다.

필수 기준:
- 관리자만 역할 변경 가능
- 관리자만 활성/비활성 변경 가능
- 관리자만 퇴출 가능
- 팀장은 팀 변경만 가능
- 팀장은 같은 팀의 활성 멤버에게만 팀 변경 가능
- 본인 자신의 역할/상태는 변경 불가

## 2. 팀 관리 권한 제한

- `POST /api/v1/teams`
- `DELETE /api/v1/teams/{teamId}`

필수 기준:
- 관리자만 팀 생성/삭제 가능
- 삭제 전 팀 소속 멤버 존재 여부 검증
- 기본 보호 팀 삭제 금지 여부 정책 확정

## 3. 리프레시 토큰 보안

현재 프론트는 리프레시 토큰 만료 시 자동 로그아웃되도록 처리되어 있습니다.

백엔드 권장 항목:
- 리프레시 토큰 회전 적용
- 재사용 감지 시 전체 세션 무효화
- 로그아웃 시 리프레시 토큰 즉시 폐기
- 만료, 위조, 재사용 케이스별 에러 코드 분리

## 4. 인증/로그인 보호

필수 또는 권장 항목:
- 로그인 실패 횟수 제한
- 일정 횟수 초과 시 지연 또는 임시 잠금
- 비활성 사용자 로그인 차단 유지
- 비활성, 퇴출, 존재하지 않는 계정 응답 정책 일관화

## 5. 근무 일정 접근 제어

- `GET /api/v1/work-schedules/all`
- `GET /api/v1/work-schedules/team/{teamId}`
- `GET /api/v1/work-schedules/me`
- `PUT /api/v1/work-schedules`
- `DELETE /api/v1/work-schedules/{dayOfWeek}`

필수 기준:
- 관리자만 전체 조회 가능
- 팀장은 본인 팀 조회만 가능
- 멤버는 본인 일정만 조회/수정 가능
- 다른 사람 일정 수정/삭제 불가

## 6. 멤버 상태 변경 이력 저장

활성화, 비활성화, 퇴출, 팀 변경, 역할 변경은 모두 감사 로그가 필요합니다.

권장 저장 항목:
- 수행자 id
- 대상자 id
- 변경 전 값
- 변경 후 값
- 변경 시각
- 사유

## 7. 입력값 검증

필수 기준:
- 팀 이름 길이 제한
- 팀 이름 공백/중복 검증
- 역할 값 화이트리스트 검증
- 시간/날짜 포맷 검증
- 문자열 필드 길이 제한

## 8. 응답 데이터 최소화

권장 기준:
- 멤버 목록 응답에 불필요한 내부 정보 제외
- 관리자 전용 필드와 일반 필드 분리
- 비활성/퇴출 사용자 노출 정책 명확화

## 9. 에러 메시지 정책

필수 기준:
- 권한 없음은 `403`
- 인증 실패는 `401`
- 존재하지 않는 리소스는 `404`
- 비즈니스 규칙 위반은 별도 코드 유지

권장 기준:
- 민감한 내부 상태를 에러 메시지에 직접 노출하지 않기

## 10. 운영 보안

권장 항목:
- CORS origin 제한
- 보안 헤더 적용
  - HSTS
  - CSP
  - X-Frame-Options
  - X-Content-Type-Options
  - Referrer-Policy
- 관리자/팀장 주요 작업 rate limit
- 관리자 기능 접근 로그 모니터링

## 우선순위 추천

가장 먼저 하면 좋은 순서:
1. 멤버/팀 관련 권한 검증 서버 강제
2. 리프레시 토큰 회전 및 재사용 감지
3. 팀 생성/삭제 권한과 검증 강화
4. 감사 로그 저장
5. 로그인 rate limit 및 잠금 정책
